# van-app — التاجر المتنقل

تطبيق React/Vite (PWA) لموظف التوصيل: تحميل بضاعة من المخزون الرئيسي، بيع للمحلات، ومحاسبة يومية.

## ⚠️ خطوة مطلوبة قبل النشر: تشغيل ملف SQL

هذا الإصدار يعتمد على دالة قاعدة بيانات جديدة `complete_van_sale` تجعل عملية
البيع **ذرّية** (atomic) بدل حلقة نداءات منفصلة كانت تعرّض المخزون لخطر
النقص عند انقطاع الشبكة في منتصف العملية.

**قبل النشر:**
1. افتح Supabase → SQL Editor
2. انسخ محتوى `supabase/complete_van_sale.sql` كاملاً وشغّله
3. تأكد أن اسم الدالة `sell_van_stock` وحقول جدول `orders` مطابقة لما لديك فعلاً (الملف يشرح كل شيء بالتفصيل)

## ما الجديد في هذا الإصدار

- بيع ذرّي: عملية بيع واحدة (خصم + تسجيل طلب) داخل معاملة واحدة بقاعدة البيانات، فلا يحدث خصم جزئي عند فشل منتصف العملية.
- دعم العمل بدون إنترنت:
  - Service Worker حقيقي (vite-plugin-pwa) يخزّن واجهة التطبيق فتُفتح حتى بدون شبكة.
  - عند فقدان الاتصال أثناء البيع، تُحفظ العملية محلياً وتُزامَن تلقائياً عند عودة الشبكة (بانر بالأعلى يوضح الحالة وعدد العمليات المعلّقة).
- حماية من الضغط المزدوج على زر "إتمام البيع"، وتمييز رسائل خطأ الشبكة عن الأخطاء المنطقية.
- Pagination في تبويب "تحميل": زر "تحميل المزيد" بدل الاقتصار على 30 نتيجة فقط.
- إضافة أيقونات PWA (public/icons/) كانت مفقودة ويشير لها manifest.json.

## التشغيل محليًا

npm install
npm run dev

يتطلب متغيرات البيئة VITE_SUPABASE_URL و VITE_SUPABASE_ANON_KEY (في .env.local محليًا، أو Environment Variables في Vercel).

## 🔐 التحقق الثنائي (OTP) عند الدخول

حالياً **وضع تجريبي**: بعد كلمة المرور، تظهر شاشة "رمز التحقق" والكود دائماً
هو **1234** — لا تُرسَل أي رسالة SMS فعلية بعد. المنطق كله معزول في
`src/lib/otp.js` بدالتين (`sendOtp`, `verifyOtp`) لتسهيل استبداله لاحقاً بخدمة
حقيقية دون تعديل أي شاشة.

**لا يوجد مزوّد SMS مجاني دائم** — كل الخيارات (Twilio, Vonage, Plivo,
Infobip...) تتقاضى عادة 0.005–0.08 دولار لكل رسالة، مع رصيد تجريبي صغير عند
التسجيل فقط. إن كنت تريد حلاً مجانياً بالكامل، البديل هو تطبيق مصادقة (TOTP)
مثل Google Authenticator بدل الرسائل النصية.

## 🖨️ الطباعة الحرارية (Bluetooth 80مم)

تبويب "محاسبة" يحتوي لوحة "🖨️ الطابعة" لاختبار الطباعة مباشرة:
1. افتح التطبيق على **Chrome على أندرويد** (Web Bluetooth لا يعمل على iOS/Safari إطلاقاً — قيد من آبل)
2. فعّل بلوتوث الهاتف وشغّل الطابعة
3. اضغط "اتصال بالطابعة" واختر جهازك من قائمة المتصفح
4. اضغط "اختبار الطباعة" وتحقق من خروج الإيصال فعلياً
5. إن خرجت الطباعة مقصوصة أو صغيرة بمنتصف الورقة، جرّب تبديل عرض الورق بين 58مم/80مم في نفس اللوحة

تُطبع الفاتورة كصورة (وليس نصاً) لضمان ظهور العربية بشكل صحيح على كل الطابعات،
بغض النظر عن دعمها للترميز العربي. بعد كل عملية بيع ناجحة، يظهر زر "طباعة
الفاتورة" في تبويب "بيع" لطباعة نفس الفاتورة يدوياً.

⚠️ لم أتمكن من اختبار هذا فعلياً على طابعة حقيقية (لا يوجد بلوتوث بهذه البيئة) —
جرّبه من جهتك وأخبرني بأي رسالة خطأ تظهر لأصلحها بدقة.

## 🛡️ التحقق الدوري من صلاحية الجلسة

⚠️ يحتاج تشغيل `supabase/check_employee_access.sql` في Supabase (نفس خطوة
complete_van_sale) — عدّل اسم الجدول/الأعمدة داخل الملف إن كانت مختلفة لديك.

بعدها، التطبيق يسأل الخادم "هل ما زال هذا الموظف مصرَّحًا له؟" عند فتح
التطبيق، كل 5 دقائق، وعند العودة له من الخلفية. لو أُلغيت صلاحية الموظف من
لوحة الإدارة، يسجَّل خروجه تلقائيًا برسالة توضيحية — دون انتظار أن يسجّل
خروجه بنفسه. لا يُسجَّل الخروج عند مجرد انقطاع الشبكة، فقط عند رفض صريح من
الخادم.

## 📱 تشغيل التطبيق كتطبيق أندرويد أصلي (Android Studio)

تم تجهيز المشروع كتطبيق أندرويد حقيقي عبر Capacitor (وليس مجرد PWA داخل متصفح).

### الخطوات

1. **جهّز متغيرات البيئة أولاً** (قبل أي بناء) — أنشئ ملف `.env.local` بجذر المشروع:
   ```
   VITE_SUPABASE_URL=https://xxxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=xxxxxxxxxxxxxxxx
   ```
   ⚠️ هذه القيم تُخبز داخل ملفات JS النهائية وقت البناء (`npm run build`)، فلو غيّرتها لاحقاً يجب إعادة البناء والمزامنة (الخطوتين التاليتين).

2. **ثبّت الاعتماديات وابنِ الواجهة:**
   ```bash
   npm install
   npm run build
   ```

3. **زامن مشروع أندرويد مع آخر بناء:**
   ```bash
   npx cap sync android
   ```

4. **افتح مجلد `android/` مباشرة من Android Studio:**
   - Open → اختر مجلد `android` (وليس جذر المشروع كله)
   - انتظر Gradle Sync (يحتاج إنترنت أول مرة لتحميل الاعتماديات من مستودعات جوجل)
   - Run ▶️ على جهاز حقيقي أو محاكي

كل مرة تُعدّل فيها كود React (أي شيء داخل `src/`)، يجب إعادة تنفيذ الخطوتين 2 و3 قبل تشغيل المشروع من Android Studio مرة أخرى، وإلا سترى نسخة قديمة.

### 🖨️ الطباعة داخل تطبيق أندرويد (مختلفة عن المتصفح)

⚠️ **مهم**: داخل تطبيق أندرويد (WebView)، لا يعمل Web Bluetooth إطلاقاً (قيد
تقني من جوجل نفسها على كل WebView، وليس خاصاً بهذا التطبيق). لذلك أضفنا مسار
طباعة منفصل خاص بالتطبيق الأصلي عبر إضافة `@capacitor-community/bluetooth-le`
(BLE)، يعمل تلقائياً بمجرد تشغيل التطبيق من Android Studio دون أي إعداد إضافي
بالكود — لوحة "🖨️ الطابعة" بتبويب المحاسبة تستخدمه تلقائياً.

**قيد مهم يجب معرفته**: هذا المسار يدعم فقط طابعات **BLE**. أغلب طابعات
الـ80مم الرخيصة تدعم BLE فعلاً (وهذا ما جرّبناه أولاً)، لكن لو كانت طابعتك من
النوع القديم الذي يدعم **Bluetooth Classic (SPP)** فقط، لن تظهر في قائمة
الاقتران داخل التطبيق. أخبرني إن واجهت هذه الحالة تحديداً وسأضيف مسار SPP
منفصل (يحتاج إضافة Capacitor مختلفة).

⚠️ لم أتمكن من اختبار هذا فعلياً على طابعة حقيقية (لا يوجد بلوتوث بهذه
البيئة) — جرّبها من جهتك عبر لوحة "🖨️ الطابعة" في تبويب "محاسبة"، وأي رسالة
خطأ تظهر انسخها لي لأصلحها بدقة.

### 📦 ما الذي تغيّر تقنياً لدعم أندرويد

- إضافة `@capacitor/core`, `@capacitor/android`, `@capacitor/cli`
- إضافة `@capacitor-community/bluetooth-le` (BLE للطباعة داخل التطبيق الأصلي)
- ملف `capacitor.config.ts` (معرّف التطبيق: `com.naqaa.vanapp`)
- مجلد `android/` — مشروع أندرويد كامل جاهز لفتحه في Android Studio مباشرة
- `src/lib/printer.js` أصبح يفرّق تلقائياً بين العمل داخل متصفح (Web Bluetooth) أو داخل التطبيق الأصلي (Capacitor BLE) دون أي تدخل يدوي
